Project 1: Asteroids

Overview

Asteroids is a classic arcade game which debuted in 1979 (see http://en.wikipedia.org/wiki/Asteroids_(video_game)). As one computer science professor describes the game, it is "a timeless tale of triumph and sacrifice, of hatred and love, of space travel wrapped around our screens and our hearts. One solitary ship, an interminable journey, a violent struggle for freedom...." Before you go any farther, stop and play the game for a few minutes: http://www.freeasteroids.org (note: the only four control keys are the space bar and up, left, and right arrows). As you play, start thinking about the "noteworthy events" of the game—things that your code will have to detect and respond to.

It's also fairly easy to implement, especially if you have help with the math. If you browse the Java API, you'll see that Java provides some classes very similar to what we need (e.g. Point and Polygon ), but for this project, we'll mostly "roll our own"—in part so that you can see the mathematics involved, in part because using the Java-provided classes in a game like this would require a more-complicated object-oriented design.

For this project, you will be required to implement the basics of the game (detailed below); there is plenty of opportunity for extra credit, implementing 'extra' features.

In this project repository, I'm providing you with some "skeleton" code. Some of it is code that you won't need to touch (or even understand)—code that provides the basic mechanics of the game graphics, for example. Some of it is code that you'll certainly need to modify and extend—I give you basic code for polygons, but how will you use that to create the ships and asteroids of the game?

You'll get a lot of answers by reading the documentation for the code, either in the files themselves or as javadocs

Java graphics

For this game (like almost any video game), you will be forced to think about the x-y coordinate plane (be glad this is a 2-D game, not a 3-D one!). You may be used to a coordinate system where the origin (0,0) is the bottom left, with positive numbers going up and to the right—or you may be familiar with a coordinate system where the origin is in the middle, with positive x to the right and positive y above. In Java, the world is a little different: the origin (0,0) is the top left corner, positive x goes to the right (as 'usual'), and positive y goes down. Angles in this system are also different than you might be used to: "0 degrees of rotation" points to the right along the positive x axis (as 'usual' in geometry/trigonometry), but increasing degrees of rotation go clockwise, so "90 degrees of rotation" is down the positive y axis, and "270 degrees of rotation" is up the negative y axis. Stop here and draw yourself a picture.

In Java, the actual drawing of things is done by a Graphics object, named brush in this code. Browse the API to see what kinds of methods are available to Graphics objects.

  • As a first step: run this skeleton code. It should produce a boring black window.
  • As a tiny second step, modify the game code to draw a white line somewhere in the center of the game display. (Where does this code go? What method(s) will you need to do this?)

Adding Behavior

Most of the 'action' will be split between Asteroid object(s) and Polygon object(s). Take a look a the Polygon code. You don't need to master the mathematics, but look at the methods available (Eclipse's outline view might be helpful). How will the game logic make use of the contains() method, for example?

OK, now

  • Declare a new class which will model your ship. It should be a subclass of Polygon
  • Modify your game code so that the ship appears in the middle of game, "pointing" upward, just like it should.
But the ship doesn't move. This next step is something of a challenge.
  • Modify your game code so that the ship starts in the center of the game and moves upward. We obviously haven't added any controls yet. Don't worry about that. (What aspects of your ship will change when it's in motion? How will you make sure that change happens uniformly across time? Looking ahead, when we add controls, your ship will need to accelerate. What are the implications of that? )
Once your ship is moving, there are two logical next steps, which you could tackle in any order you please:
  • Populate the game with at least a couple asteroids. What about their motion?
  • Add controls to the ship; see below

Adding Control

To get our ship to respond to key-presses, it needs to implement an interface. In particular, it needs to implement the KeyListener interface. Browse the Java API— which package is this in? What methods do you need to implement in order to correctly implement this interface? (Hint: you must provide an implementation of KeyTyped() , but your implementation should be empty. Let the other methods do the work.)

Decide which three keys you want to use to control the ship. (Hint: in my experience, it's easier to work with the alphanumeric keys than with the arrow keys.)

  • Implement the KeyListener interface so that it "hears" and responds to your control keys. Hint: just set boolean variables indicating which key has been pressed/released. Don't change the ship's behavior in the interface implementation; instead, test the boolean variables in the part of your code that affects the ship's movement.
  • Once the interface is implemented, modify your game code so that your ship responds properly to your controls.

Interaction

Once you can control your ship, it's fair to expect you to avoid those asteroids!

  • Modify your game code so that something "appropriate" happens if your ship collides with an asteroid.

Doing More

You've now done everything you need to do to earn 100% on this project. Yes, there is lots more that could be done! For extra credit (plus your personal satisfaction), you might want to:

  • Give your ship the ability to shoot bullets.
  • Give those bullets the power to degrade/destroy asteroids.
  • Give your asteroids the ability to split into smaller asteroids.
  • Give the game the ability to keep the number of asteroids on the board constant (so if one is destroyed, a new one 'spawns' soon after)
  • Or, have the number of asteroids increase over time, increasing the difficulty of the game
  • Add stars in the 'background.' Your ship and the asteroids should be "in front of" the stars.
  • Make the stars 'twinkle.'
  • Give your ship multiple 'lives.'
  • Display a running score.
  • Anything else you can dream of!
